## Changes Made

### 1 — Fix camelCase JSON tags on domain models (plan-and-implement)
**Problem:** Go's `encoding/json` defaults to PascalCase for untagged struct fields, but DataTable column keys are camelCase (`paidByName`, `convertedAmount`). This caused `row[column.key]` to always return `undefined`, rendering all table cells as `"-"` or broken action links.
**Files:** `internal/domain/models.go`, `internal/web/handlers_api_test.go`
**Change:**
- Added explicit camelCase `json` tags to all domain structs: `Expense`, `User`, `Preferences`, `CategorySummary`, `DailyGroup`
- Removed `json:"-"` from `ConvertedAmount` so currency renderer works
- Unified legacy snake_case tags (`paid_by_id` → `paidById`, `paid_by_name` → `paidByName`)
- Updated `TestAPIGet_Success` to assert `m["id"]` instead of `m["ID"]`
**Tests:** 28 Go tests pass, 7/7 frontend tests pass

### 2 — Fix `default` template function parameter order
**Problem:** `{{ .variant | default "default" }}` passes the piped value as the **last** argument, but the function was `func(v, def)`. This meant `v` always received the fallback string, so ghost variant was never applied.
**Files:** `internal/web/renderer.go`
**Change:** Swapped parameters to `func(def, v)` to match Go template pipe convention.
**Tests:** All pass

### 3 — Fix `humanDate` timezone bug
**Problem:** `time.Parse("2006-01-02", ...)` returns UTC midnight, while `time.Now().Truncate(24h)` returns local midnight. At GMT+0800, comparing them produced incorrect day diffs ("Today" showed as "6 hours from now").
**Files:** `internal/web/renderer.go`
**Change:** Replaced time-math comparison with direct string comparison of `"YYYY-MM-DD"` dates. Also fixed `formatDate` to use `time.ParseInLocation(..., time.Local)`.
**Tests:** All pass

### 4 — Add `formatDate` template helper

**Files:** `internal/web/renderer.go`, `templates/daily.html`

## Decisions & Rationale
1. **camelCase everywhere for JSON** — the React/JS side expects it, it's the de facto JSON standard, and the old snake_case tags on `PaidByID`/`PaidByName` were inconsistent with the rest of the codebase.
2. **Tag all domain structs** — not just `Expense` and `User`. Prevents this class of bug entirely and makes API responses predictable.
3. **Direct string comparison for dates** — simpler, faster, and timezone-proof. Time-math with mixed UTC/local was a latent bug that only surfaced in non-UTC timezones.
4. **Icon in button partial, not separate component** — avoids adding a new abstraction when the partial already handles all button cases. `safeHTML` is a reusable escape hatch for other inline SVGs too.
