### Bug found during self-grill (and fixed)
**Issue:** `json` template func returned `template.HTML`, bypassing `html/template` escaping. A single quote in user data (e.g., `"It's great"` in an expense description) would break out of `data-props='{{json .Data}}'` — both XSS vector and runtime breakage.

**Fix:** Changed return type from `template.HTML` to `string`. `html/template` now HTML-escapes the JSON; browsers decode entities when reading `dataset.props`; `JSON.parse()` works correctly.
