### 2 ‚Äî GitHub Actions CI for Docker Builds
**Problem:** User's local machine is arm64 (Apple Silicon), Unraid is amd64. Building locally and pushing to GHCR resulted in platform mismatch warning.
**Files:** `.github/workflows/docker.yml`
**Change:**
- Workflow triggers on push to `main` and semver tags
- Builds `linux/amd64` only (per user request)
- Pushes to `ghcr.io/johngohrw/expensif`
- Tags: `main`, semver, short SHA
- Uses `GITHUB_TOKEN` with `packages: write` permission
- Uses GitHub Actions cache for Docker layer caching
**Issue encountered:** First push failed with `write_package` denied. Root cause: GHCR package was created by local `docker push` under user account, Actions token lacked access. **Fix:** Added repository write access in package settings (`https://github.com/users/johngohrw/packages/container/expensif/settings`).

### 3 ‚Äî Air Hot Reload for Local Development
**Problem:** `make dev` required manual restart of Go server when `.go` files changed.
**Files:** `.air.toml`, `Makefile`, `.gitignore`
**Change:**
- Created `.air.toml` with sensible defaults for the project structure
- `full_bin = "DEV=true ./tmp/main"` ‚Äî enables dev mode automatically
- Excludes `ui/`, `static/`, `.context/`, `.skills/`, `.git/` from watcher
- Ignores `_test.go` files
- Updated `Makefile` `dev` target to use `air` instead of `go run`
- Added `tmp/` to `.gitignore`
**Issue encountered:** `air: command not found` ‚Äî binary was at `~/go/bin/air` but `~/go/bin` wasn't in `$PATH`. **Fix:** Added `export PATH="$HOME/go/bin:$PATH"` to `~/.zshrc`.

## Decisions & Rationale
1. **`DATA_DIR` env var** ‚Äî Makes the container portable across Docker Compose, bare metal, and Unraid without hardcoding paths. Clean separation of data from application code.
2. **amd64-only builds** ‚Äî User explicitly requested. Can be expanded to multi-platform (`linux/arm64,linux/amd64`) later if needed for local Mac development.
3. **Air for hot reload** ‚Äî Much faster than manual restart. `make dev` still works identically, just auto-rebuilds Go code on save.
4. **GHCR over Docker Hub** ‚Äî Natural fit for GitHub-hosted code, no separate account, free tier is generous.

## Unraid Deployment Reference
**Container settings:**

**Important:** Ensure the GHCR package is **public** (Settings ‚Üí Package ‚Üí Make public) or Unraid cannot pull without authentication.
